# m-index-pages

## 项目简介

`m-index-pages` 是一个用于归档和发布多个独立 HTML 页面的仓库。  
本仓库不针对任何特定域名或网站，仅作为静态 HTML 文件的集合存储与展示空间。

## 目录说明

```
m-index-pages/
├── pages/          # 存放各独立 HTML 页面
├── assets/         # 页面引用的公共资源（如 CSS、JS、图片等）
└── README.md       # 本文件
```

- `pages/`：每个子目录或文件对应一个独立页面，命名清晰，便于识别。
- `assets/`：存放页面间共享的样式、脚本及媒体资源，避免重复引用。

## 页面归档说明

本仓库中的 HTML 页面均为独立归档，互不依赖。  
每个页面可单独通过其路径访问，无需额外配置。  
页面内容可能包含静态信息、演示、工具或存档资料，具体用途请参考各页面内说明。

## 维护说明

- 新增页面：在 `pages/` 下创建新目录或文件，并确保引用资源路径正确。
- 更新页面：直接修改对应 HTML 文件，保持结构清晰。
- 删除页面：移除相关文件及引用资源，避免遗留无效链接。
- 资源管理：公共资源放置于 `assets/`，避免重复上传。

请确保提交的 HTML 文件符合基本规范，不包含恶意代码或外部依赖。
